## 新增需求

### 需求：建议列表展示
Review 页面必须以卡片形式展示所有待审批建议，包含类型、描述、置信度和频率。

#### 场景：按类型分组显示
- **当** 用户访问 Review 页面
- **则** 建议按类型分组（Preferences、Patterns、Workflows）
- **则** 每个分组显示该类型的建议数量
- **则** 分组可折叠/展开

#### 场景：建议卡片显示关键信息
- **当** 用户查看建议列表
- **则** 每张卡片显示类型图标、描述前 100 字符
- **则** 显示置信度进度条和频率数字
- **则** 卡片右侧显示批准/拒绝按钮

### 需求：建议详情展开
Review 页面必须支持展开建议查看完整详情和证据引用。

#### 场景：展开查看完整信息
- **当** 用户点击建议卡片或"View Details"按钮
- **则** 卡片展开显示完整描述
- **则** 显示所有 evidence 条目（前 5 条 + "还有 N 条"）
- **则** 显示创建时间和完整 UUID

#### 场景：折叠隐藏详情
- **当** 用户点击已展开卡片的"Collapse"按钮
- **则** 卡片折叠回紧凑视图
- **则** 仅显示摘要信息

### 需求：单个建议操作
Review 页面必须支持对单个建议进行批准或拒绝操作。

#### 场景：批准建议
- **当** 用户点击建议的"Approve"按钮
- **则** 发送 POST /api/suggestions/:id/approve 请求
- **则** 按钮显示加载状态
- **则** 成功后建议从列表中移除并显示成功提示

#### 场景：拒绝建议
- **当** 用户点击建议的"Reject"按钮
- **则** 显示确认对话框
- **则** 用户确认后发送 POST /api/suggestions/:id/reject
- **则** 成功后建议从列表中移除

### 需求：批量选择和操作
Review 页面必须支持批量选择建议并执行批量操作。

#### 场景：选择多个建议
- **当** 用户勾选建议卡片左上角的复选框
- **则** 建议被标记为选中状态
- **则** 底部固定栏显示"N selected"

#### 场景：批量批准
- **当** 用户选择多个建议后点击"Approve All"
- **则** 发送 POST /api/suggestions/batch/approve 带选中的 IDs
- **则** 所有成功的建议从列表移除
- **则** 显示批量操作结果摘要

### 需求：搜索和筛选
Review 页面必须提供搜索和筛选功能以快速定位建议。

#### 场景：按类型筛选
- **当** 用户点击顶部的类型标签（All / Preferences / Patterns / Workflows）
- **则** 列表仅显示匹配类型的建议
- **则** 标签高亮表示当前筛选

#### 场景：按关键词搜索
- **当** 用户在搜索框输入关键词
- **则** 列表实时过滤显示描述中包含关键词的建议
- **则** 搜索不区分大小写

### 需求：排序选项
Review 页面必须支持按不同维度对建议列表排序。

#### 场景：按置信度排序
- **当** 用户选择"Sort by Confidence"
- **则** 建议按置信度从高到低排列

#### 场景：按时间排序
- **当** 用户选择"Sort by Latest"
- **则** 建议按创建时间从新到旧排列
