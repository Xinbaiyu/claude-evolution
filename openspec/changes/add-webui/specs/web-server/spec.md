## 新增需求

### 需求：Web 服务器初始化
系统必须在执行 `claude-evolution start` 时启动 HTTP 服务器，并绑定到可配置的端口。

#### 场景：服务器启动成功
- **当** 用户运行 `claude-evolution start`
- **则** HTTP 服务器绑定到配置的端口（默认 10010）
- **则** 控制台显示 "Web server running at http://localhost:10010"

#### 场景：端口已被占用
- **当** 配置的端口已被其他进程占用
- **则** 系统显示包含被占用端口号的错误消息
- **则** 系统建议使用其他端口或修改配置

### 需求：静态资源服务
系统必须通过 Web 服务器提供 React 前端构建产物。

#### 场景：根路径返回 index.html
- **当** 用户访问 http://localhost:10010/
- **则** 服务器返回 React 构建的 index.html
- **则** 响应包含正确的 Content-Type 头

#### 场景：静态资源缓存
- **当** 浏览器请求 JS/CSS 资源
- **则** 服务器设置 Cache-Control 头以实现高效缓存
- **则** 带内容哈希的版本化资源使用长期缓存策略

### 需求：WebSocket 服务器初始化
系统必须在 HTTP 服务器旁启动 WebSocket 服务器用于实时通信。

#### 场景：WebSocket 连接建立
- **当** React 前端连接到 ws://localhost:10010
- **则** WebSocket 握手成功完成
- **则** 连接被注册以接收通知

#### 场景：心跳保持连接
- **当** WebSocket 连接空闲 30 秒
- **则** 服务器发送 ping 帧
- **则** 客户端响应 pong 帧以保持连接活跃

### 需求：优雅关闭
系统必须在停止 Web 服务器时干净地关闭所有连接。

#### 场景：服务器响应 SIGTERM
- **当** 进程接收到 SIGTERM 信号
- **则** 服务器停止接受新连接
- **则** 现有 WebSocket 连接收到关闭帧
- **则** HTTP 服务器在所有请求完成后关闭
