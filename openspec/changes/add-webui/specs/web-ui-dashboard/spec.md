## 新增需求

### 需求：系统状态卡片显示
Dashboard 页面必须显示包含调度器状态、上次分析时间和下次计划时间的状态卡片。

#### 场景：显示调度器启用状态
- **当** 用户访问 Dashboard 页面
- **则** 页面显示绿色指示器和"Scheduler Active"文本
- **则** 显示上次分析的相对时间（如"2 hours ago"）

#### 场景：显示调度器禁用状态
- **当** 调度器被配置为禁用
- **则** 页面显示灰色指示器和"Scheduler Inactive"文本
- **则** 显示"立即分析"按钮以手动触发

### 需求：指标卡片显示
Dashboard 页面必须显示待审批、已批准和平均置信度的指标卡片。

#### 场景：显示待审批数量
- **当** 用户访问 Dashboard
- **则** 显示当前待审批建议数量
- **则** 数量大于 0 时使用黄色强调色
- **则** 点击卡片跳转到 Review 页面

#### 场景：显示已批准统计
- **当** 用户访问 Dashboard
- **则** 显示累计已批准建议数量
- **则** 使用绿色强调色

#### 场景：显示平均置信度
- **当** 用户访问 Dashboard
- **则** 显示所有待审批建议的平均置信度百分比
- **则** 置信度高于 80% 时使用绿色，低于 60% 时使用黄色

### 需求：最近建议列表
Dashboard 页面必须显示最近生成的 5 条待审批建议的紧凑列表。

#### 场景：显示建议摘要
- **当** 用户访问 Dashboard
- **则** 显示最新 5 条建议的类型、描述和置信度
- **则** 每条建议显示快捷批准/拒绝按钮
- **则** 底部显示"查看全部"链接到 Review 页面

### 需求：快捷操作按钮
Dashboard 页面必须提供快捷操作按钮以触发常用功能。

#### 场景：立即分析按钮
- **当** 用户点击"Analyze Now"按钮
- **则** 触发 POST /api/analyze 请求
- **则** 按钮显示加载状态
- **则** 分析开始后显示成功提示

### 需求：实时数据更新
Dashboard 页面必须在收到 WebSocket 通知时自动刷新数据。

#### 场景：分析完成后自动刷新
- **当** WebSocket 推送"analysis_complete"事件
- **则** Dashboard 重新获取指标数据
- **则** 指标卡片数字平滑动画更新
- **则** 最近建议列表自动刷新
