## 新增需求

### 需求：已批准建议列表
History 页面必须显示所有已批准建议的列表，包含批准时间和应用状态。

#### 场景：显示已批准历史
- **当** 用户访问 History 页面
- **则** 显示所有 approved.json 中的建议
- **则** 每条记录显示类型、描述、批准时间
- **则** 按批准时间倒序排列

#### 场景：显示应用状态
- **当** 建议已应用到 CLAUDE.md 文件
- **则** 显示绿色"Applied"标签
- **则** 显示应用到的文件路径

### 需求：已拒绝建议列表
History 页面必须显示所有已拒绝建议的列表。

#### 场景：显示已拒绝历史
- **当** 用户切换到"Rejected"标签
- **则** 显示所有 rejected.json 中的建议
- **则** 每条记录显示拒绝时间和原因（如有）

### 需求：时间线可视化
History 页面必须提供时间线视图展示分析活动历史。

#### 场景：显示活动时间线
- **当** 用户切换到"Timeline"视图
- **则** 显示按日期分组的分析活动
- **则** 每个日期节点显示分析会话数、生成的建议数
- **则** 点击节点展开该日期的详细活动

#### 场景：日期范围筛选
- **当** 用户选择日期范围
- **则** 时间线仅显示范围内的活动
- **则** 统计数据相应更新

### 需求：搜索历史记录
History 页面必须提供搜索功能以快速定位历史建议。

#### 场景：按关键词搜索
- **当** 用户在搜索框输入关键词
- **则** 列表过滤显示描述或类型匹配的记录
- **则** 搜索跨越已批准和已拒绝两个列表

### 需求：导出历史数据
History 页面必须支持导出历史记录为 JSON 或 CSV 格式。

#### 场景：导出为 JSON
- **当** 用户点击"Export"并选择 JSON 格式
- **则** 下载包含所有历史记录的 JSON 文件
- **则** 文件名包含日期时间戳

#### 场景：导出为 CSV
- **当** 用户选择 CSV 格式
- **则** 下载包含建议关键字段的 CSV 文件
- **则** 可在 Excel 中打开查看
