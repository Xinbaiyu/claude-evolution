## 新增需求

### 需求：后台进程启动
系统必须支持以守护进程模式启动调度器和 Web 服务器。

#### 场景：启动后台服务
- **当** 用户运行 `claude-evolution start`
- **则** 进程在后台启动（不阻塞终端）
- **则** 调度器和 Web 服务器同时初始化
- **则** 进程 ID 保存到 ~/.claude-evolution/daemon.pid

#### 场景：阻止重复启动
- **当** daemon.pid 文件存在且进程运行中
- **则** 系统显示"Server is already running"错误
- **则** 显示当前运行的进程 ID 和 Web URL

### 需求：后台进程停止
系统必须支持停止正在运行的守护进程。

#### 场景：停止后台服务
- **当** 用户运行 `claude-evolution stop`
- **则** 系统读取 daemon.pid 文件
- **则** 向进程发送 SIGTERM 信号
- **则** 等待进程优雅关闭
- **则** 删除 daemon.pid 文件

#### 场景：进程不存在时停止
- **当** daemon.pid 中的进程不存在
- **则** 系统显示"No daemon process found"
- **则** 清理 daemon.pid 文件

### 需求：后台进程状态查询
系统必须提供查询守护进程状态的命令。

#### 场景：查询运行中的服务
- **当** 用户运行 `claude-evolution status`
- **则** 显示进程状态（运行中/已停止）
- **则** 显示进程 ID、启动时间
- **则** 显示 Web 服务器 URL 和端口
- **则** 显示调度器状态和下次分析时间

#### 场景：查询已停止的服务
- **当** 守护进程未运行
- **则** 显示"Daemon is not running"
- **则** 提示运行 `claude-evolution start` 启动

### 需求：进程日志管理
系统必须记录守护进程的运行日志。

#### 场景：写入日志文件
- **当** 守护进程运行时
- **则** stdout/stderr 重定向到 ~/.claude-evolution/logs/daemon.log
- **则** 日志包含时间戳和日志级别
- **则** 日志文件自动轮转（超过 10MB 时）

#### 场景：查看日志
- **当** 用户运行 `claude-evolution logs`
- **则** 显示最近 50 行日志
- **则** 支持 --follow 参数实时跟踪日志

### 需求：进程崩溃恢复
系统必须在进程意外退出时记录状态。

#### 场景：记录崩溃信息
- **当** 守护进程因错误退出
- **则** 错误堆栈写入 crash.log
- **则** daemon.pid 文件被清理
- **则** 下次启动时检测到崩溃并显示提示
