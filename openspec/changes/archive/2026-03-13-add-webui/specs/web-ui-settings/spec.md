## 新增需求

### 需求：调度器配置界面
Settings 页面必须提供调度器启用/禁用开关和间隔配置。

#### 场景：切换调度器状态
- **当** 用户点击"Enable automatic analysis"开关
- **则** 发送 PATCH /api/config 更新 scheduler.enabled
- **则** 开关状态立即切换
- **则** 显示"配置已保存"提示

#### 场景：修改分析间隔
- **当** 用户选择间隔选项（6h / 12h / 24h / Custom）
- **则** 下拉菜单更新选中状态
- **则** 选择 Custom 时显示 cron 表达式输入框
- **则** 保存后调度器使用新间隔

### 需求：LLM 配置界面
Settings 页面必须提供 LLM 模型选择、token 限制和温度配置。

#### 场景：选择 LLM 模型
- **当** 用户从下拉菜单选择模型
- **则** 显示可用模型列表（haiku、sonnet、opus）
- **则** 保存后分析使用新模型

#### 场景：调整最大 Tokens
- **当** 用户在输入框修改 maxTokens 值
- **则** 输入验证确保值在合理范围（1000-100000）
- **则** 保存后生效

#### 场景：调整温度参数
- **当** 用户拖动温度滑块
- **则** 滑块显示当前值（0.0-1.0）
- **则** 保存后分析使用新温度

### 需求：通知配置界面
Settings 页面必须提供桌面通知和事件订阅配置。

#### 场景：启用桌面通知
- **当** 用户勾选"Desktop notifications"
- **则** 浏览器请求通知权限（如未授权）
- **则** 保存后分析完成时显示系统通知

#### 场景：配置通知事件
- **当** 用户勾选"Show when analysis completes"
- **则** 分析完成时触发桌面通知
- **则** 通知内容包含新建议数量

### 需求：学习阶段配置
Settings 页面必须提供学习阶段启用和持续时间配置。

#### 场景：修改观察期天数
- **当** 用户修改 Observation 阶段的 durationDays
- **则** 输入验证确保为正整数
- **则** 保存后新行为数据收集周期更新

#### 场景：调整自动应用阈值
- **当** 用户修改 Automatic 阶段的 confidenceThreshold
- **则** 滑块显示百分比值
- **则** 保存后高于该阈值的建议自动应用

### 需求：API 配置界面
Settings 页面必须提供 API 模式选择和自定义端点配置。

#### 场景：切换到 Router 模式
- **当** 用户选择"Router"单选按钮
- **则** 显示 Base URL 输入框
- **则** 保存后 API 请求路由到自定义端点

#### 场景：测试连接
- **当** 用户点击"Test Connection"按钮
- **则** 发送测试请求到配置的 API 端点
- **则** 显示连接成功或失败提示

### 需求：配置保存和重置
Settings 页面必须提供保存配置和恢复默认值功能。

#### 场景：保存所有配置
- **当** 用户点击"Save Changes"按钮
- **则** 所有修改的配置批量保存到 config.json
- **则** 显示保存成功提示
- **则** 相关服务重新加载配置

#### 场景：恢复默认配置
- **当** 用户点击"Reset to Defaults"按钮
- **则** 显示确认对话框
- **则** 确认后所有配置恢复为初始默认值
- **则** 页面刷新显示默认配置
