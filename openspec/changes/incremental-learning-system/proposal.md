# Incremental Learning System with Temporal Decay

## Why

当前 claude-evolution 的学习系统存在以下问题：

1. **批量审批负担重** - 每次分析可能生成上百条建议，用户需要逐一审核
2. **学习周期长** - 必须等待观察期 + 建议期才能应用规则
3. **缺乏遗忘机制** - 旧的、不再相关的建议会永久累积
4. **置信度静态** - 一旦生成建议，置信度不再变化

受 OpenClaw 的 Retain/Recall/Reflect 和 Temporal Decay 机制启发，我们设计一个**增量学习系统**，让 Agent 能够：
- 持续累积观察
- 自动合并和去重
- 基于时间衰减遗忘旧知识
- 高置信度知识自动应用

## What

### 核心机制

#### 1. 每 6h 增量分析 + LLM 合并

```
每6h调度 →
  [旧 observations] + [新会话分析] →
    LLM去重合并 →
      更新置信度 →
        应用时间衰减 →
          删除低价值 →
            自动晋升高置信度 →
              重新生成 CLAUDE.md
```

**关键点**：
- 每次都让 LLM 看到全部旧观察（不是增量追加）
- LLM 负责去重、合并、更新置信度
- 删除机制保证数据量稳定在目标范围（如 50 条）

#### 2. 时间衰减（Temporal Decay）

```typescript
// 公式
decayedConfidence = originalConfidence × e^(-λ × ageInDays)
λ = ln(2) / halfLifeDays

// 默认：30天半衰期
```

**效果**：
- 今天：100%
- 30天前：50%
- 90天前：12.5%
- 长期未被观察到的知识自然淡化

#### 3. 自动晋升机制

**三级阈值**：
- 🥇 **自动晋升到上下文**：confidence ≥ 90% 且 mentions ≥ 10
- 🥈 **高优先级待审核**：confidence ≥ 75% 且 mentions ≥ 5
- 🥉 **候选区观察**：confidence ≥ 60% 且 mentions ≥ 3

**自动删除阈值**：
- confidence < 25%：立即删除
- confidence < 35% 且 14天未增长：延迟删除

### 数据分层

```
~/.claude-evolution/
  memory/
    observations/
      active.json          # 候选区（目标：50条）
      context.json         # 已晋升到上下文（自动应用）

  learned/                 # 最终生成的规则文件
    preferences.md
    patterns.md
    workflows.md

  output/
    CLAUDE.md              # 合成最终配置
```

#### 手动操作增强

- **批准** → `inContext: true`, 强制设置 `confidence: 95%, mentions: 20`
- **拒绝** → 立即删除（不进入候选区）
- **忽略** → 保留在候选区，但降低优先级

## Success Criteria

1. **数据量可控** - 候选区观察数稳定在配置目标（默认 50±10）
2. **自动化率提升** - 80%+ 的高置信度规则自动晋升，无需人工审批
3. **审批负担降低** - 用户每周仅需审核 5-10 条中等置信度建议
4. **知识新鲜度** - 90天未观察到的模式自动衰减到 < 15%
5. **成本可控** - LLM token 消耗 < 100k tokens/天（约 $0.15/天）

## Out of Scope

- 监控面板（仅设计思路，后续实现）
- 多语言模型支持（仅使用配置的默认 LLM）
- 图形化证据链可视化（后续迭代）
- 分布式学习（单机版足够）
